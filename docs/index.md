## Python is the instrument, not the result

A script can run and still answer the wrong question. A chart can look polished
and still hide missing data. A model can score well because the answer leaked
into its inputs.

This course teaches the habits that prevent those failures. Each lesson asks
you to frame a decision, predict what should happen, build one small step,
check it aggressively, and explain the boundary of the result.

## The learning path

| Stage | Lessons | What changes in your work |
| --- | --- | --- |
| **Frame** | 01–03 | Questions, values, and functions become explicit contracts. |
| **Shape** | 04–06 | Arrays and tables keep their axes, keys, and quality rules. |
| **Understand** | 07–08 | Plots and statistics describe evidence with uncertainty. |
| **Model** | 09–10 | Baselines, splits, and metrics keep evaluation honest. |
| **Deliver** | 11–13 | A tested project becomes a one-command reproducible report. |

The capstone uses a small synthetic customer-renewal dataset included in the
repository. There are no downloads, accounts, or hidden services required.
